MockQA

Functional Testing in minutes

Are you a developer using jQuery? With MockQA you can write and run functional tests for your projects in a couple of minutes.

How?

Get MockQA

You can either download the ZIP or clone directly from github.

Write tests

Write tests using plain English and CSS selectors, including jQuery's selector extensions. 

type           [name="firstname"]     Bruce
click          :submit
assertText     #greeting              Hi, Bruce!

Run tests

Tests can be run in any browser on any device. No installers, no plugins.

Why?

MockQA is designed to quickly let you automate the manual testing that gets in the way of getting things done.

  • It allows you to write tests in minutes.
  • It can be distributed with projects and will just work. No installers, no plugins.
  • It automatically waits for asynchronous processes and page loads to complete. No wait() statements, ever.
  • It lets you run tests in actual browsers and use built-in dev tools to inspect results.

MockQA is as much a developer productivity tool as it is a functional testing tool. There are a lot of JavaScript testing frameworks and tools, but MockQA is not intended to replace any of them. It is intended to make your life as a developer a little more fun by automating the parts that are boring.

Demos

Backbone Test

TodoMVC is a standardized example app for JavaScript MVC frameworks. This demo tests every interactive component of the Backbone example in a couple of seconds.

jQuery UI Test

jQuery UI is the widgets and effects library for jQuery. This demo shows how MockQA can test widgets like tabs, autocompletes and accordions.

Forms & links Test

This demo shows functional tests for standard form elements like text inputs, selects, radiobuttons and checkboxes. The demo also shows that you can have one test run across multiple pages.

Ajax & mocking Test

Mockjax is a jQuery plugin by Jonathan Sharp that allows you to intercept and mock Ajax requests from your code. This is very useful during debugging, development and, yes, testing. This demo shows how to configure mockjax from within a MockQA test.
