How to run a MockQA test

Now that you know how to write a test you need to run it. To do this, you need to take 4 steps:

  1. Save the test file.
  2. Write a testlist that binds tests to HTML files.
  3. Include MockQA in your HTML.
  4. Open the HTML page in a browser and choose the test from the MockQA menu.

Saving test files

Let's assume your project's directory structure looks like this after downloading MockQA:

▶ myproject
    ▶ mockqa
    index.html

Now, you have to create a directory where MockQA can find test files. You can use whatever name you like and put it wherever you like (as long as it's accessible through HTTP) but I recommend calling it mockqa_tests and putting it in the root of your project.

▶ myproject
    ▶ mockqa
    ▶ mockqa_tests
    index.html

Now, save your test in this directory. For example, let's call it login_test.

▶ myproject
    ▶ mockqa
    ▶ mockqa_tests
        login_test
    index.html

Writing a test listing

MockQA needs to know which tests should test which pages. To specify this, you need to add a file called testlist to the mockqa_tests directory:

▶ myproject
    ▶ mockqa
    ▶ mockqa_tests
        testlist
        login_test
    index.html

To be able to run login_test on index.html, open this testlist file in a text editor and enter this:

login_test      index.html

Include MockQA

To include MockQA, add the following line to your HTML right before the closing </body> tag.

<script data-tests="mockqa_tests" src="mockqa/mockqa.js"></script>

The data-tests attribute should contain the path to the directory containing the MockQA tests and testlist.

Open the HTML and run the test

If you open your HTML file in a browser you should see something like this on the right side of the page. This example has several tests specified for several HTML files.

  • To run a test, click its name in the menu.
  • To run all tests for an HTML file in a row, click it's name.
  • To run all tests in a row, click the RUN ALL button.
