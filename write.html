How to write a MockQA test

MockQA tests are written in a custom format designed to be easy to write and easy to read. For example, here's what a test could look like for a simple login form:

# First, enter incomplete credentials
type               #username           johndoe
click              #submit
assertText         .errorMessage       "Please fill in your password"

# Then, enter wrong credentials
type               #password           notmypassword
click              #submit
assertText         .errorMessage       "Username or password incorrect"

# Then, enter correct credentials
type               #password           myactualpassword
click              #submit
assertHidden       .errorMessage
assertText         #welcome            Welcome, John Doe!

Let's go over this line by line:

# First, enter incomplete credentials

Line's starting with # are comments. Their contents are completely ignored.

type               #username           johndoe

This is a simulated user action. In this case it simulates pressing the keys to type "johndoe" in the element with id="username".

click              #submit

You probably already guessed that this simulates a left mouse click on the element with id="submit".

assertText         .errorMessage       "Please fill in your password"

This line checks if there is an element with class="errorMessage" that contains the text "Please fill in your password".

API

There are a couple of more actions you can simulate besides click and type as well as quite a number of asserts in addition to assertText. You can find them in the API documentation.
